Documentation technique du smart contract OracleProtocol — structure, entrypoints, mécanique ELO et tests de sécurité.

Qu'est-ce que ce contrat fait ?

OracleProtocol est un marché de prédiction sur Tezos. Des utilisateurs (les "Oracles") parient des XTZ sur l'issue d'une question. Le gagnant reçoit une part de la cagnotte proportionnelle à son poids — lui-même calculé à partir de sa mise, de son niveau de confiance et de son score de réputation ELO.

Les 4 actions possibles

1
create_prediction — L'admin crée une question avec une deadline
Ex : "Le prix du XTZ dépassera $2 avant le 1er juin ?"
2
place_bet — Un Oracle mise des XTZ avec un niveau de confiance (50%, 75% ou 95%)
2% sont prélevés automatiquement pour la bonus pool
3
resolve_prediction — L'admin désigne le résultat gagnant et déclenche les calculs ELO
Chaque pari gagnant reçoit un weight = net_amount × confiance × ELO
4
claim_reward — Le gagnant réclame sa part de la pool
reward = (son weight / total des weights gagnants) × pool nette

Variables globales du contrat

adminaddress — seul compte autorisé à résoudre les prédictions oracle_addressaddress — alias de admin (résolveur) prediction_countnat — compteur auto-incrémenté, sert d'ID bonus_poolmutez — cagnotte alimentée par les 2% de frais elo_scoresmap[address → int] — score de réputation par wallet predictionsmap[nat → Prediction] — toutes les prédictions

Structure d'une Prediction

creatoraddress descriptionstring deadlinetimestamp — plus de paris après cette date optionslist[string] — ex: ["Oui", "Non"] statusnat — 0=OPEN, 1=CLOSED, 2=RESOLVED winning_optionoption[nat] — None jusqu'à la résolution total_poolmutez — somme des mises nettes total_winning_weightnat — Σ weights des gagnants betsmap[nat → Bet] — paris indexés bet_countnat — nombre de paris

Structure d'un Bet

bettoraddress — wallet du parieur choicenat — index dans la liste options amountmutez — mise nette (après 2% fee) confidencenat — valeur parmi {50, 75, 95} weightnat — calculé à la résolution (0 si perdant) claimedbool — False → True après claim_reward
entrypoint create_prediction admin seulement

Crée une nouvelle prédiction ouverte aux paris. Aucune restriction sur le sender dans le code actuel — c'est une note de conception : l'admin est le seul à appeler cette fonction par convention.

Paramètres

descriptionstring — texte de la question deadlinetimestamp — date limite des paris optionslist[string] — liste des choix possibles

Ce que fait le contrat

1
Crée un record Prediction avec status=OPEN, pools à 0, bets vide
2
L'insère dans predictions[prediction_count]
3
Incrémente prediction_count
L'ID de la prédiction = valeur avant incrémentation
entrypoint place_bet tout utilisateur

Permet à un Oracle de placer un pari sur une prédiction ouverte en envoyant des XTZ avec la transaction.

Paramètres

prediction_idnat — ID de la prédiction visée choicenat — index de l'option choisie confidencenat — obligatoirement 50, 75 ou 95

Validations (assert)

!
confidence ∈ {50, 75, 95} → sinon INVALID_CONFIDENCE
!
prediction_id < prediction_count → sinon INVALID_ID
!
now < deadline → sinon DEADLINE_PASSED
!
status == OPEN → sinon NOT_OPEN

Calcul des frais

fee = amount × 2 / 100 (sp.split_tokens) net_bet = amount - fee bonus_pool += fee total_pool += net_bet

Exemple : 10 XTZ misés → fee = 0.2 XTZ → net = 9.8 XTZ enregistrés dans le pari

entrypoint resolve_prediction oracle_address seulement

Désigne le résultat gagnant, calcule le weight de chaque pari gagnant, et met à jour le score ELO de tous les parieurs.

Ordre des opérations — important

1
Vérifie sender == oracle_address, sinon NOT_AUTHORIZED
2
Vérifie status == OPEN, sinon ALREADY_RESOLVED
3
Passe status → RESOLVED et enregistre winning_option
4
Boucle sur tous les paris — pour chacun :
a) Initialise ELO à 1000 si premier pari du wallet
b) Si gagnant → calcule weight avec l'ELO AVANT mise à jour
c) Met à jour ELO ± delta selon confiance

Formule du weight

current_elo = elo_scores[bettor] ← valeur AVANT update amt_nat = bet.amount en mutez / 1 weight = amt_nat × confidence × current_elo total_winning_weight += weight
Note design : le weight utilise l'ELO avant la mise à jour. Un gagnant avec 95% de confiance voit son ELO augmenter de 60 points APRES le calcul de son weight.

Mise à jour ELO (paliers fixes)

Confiance 50% → delta = ±15 points Confiance 75% → delta = ±30 points Confiance 95% → delta = ±60 points Victoire : new_elo = current_elo + delta Défaite : new_elo = current_elo - delta Initial : 1000 (si première apparition)
entrypoint claim_reward gagnant seulement

Permet au gagnant d'un pari de récupérer sa récompense. Envoie les XTZ directement sur son wallet.

Paramètres

prediction_idnat — ID de la prédiction bet_idnat — index du pari dans predictions[id].bets

Validations (dans l'ordre)

!
status == RESOLVED → NOT_RESOLVED
!
bet.bettor == sp.sender → NOT_YOUR_BET
Empêche de voler le gain d'un autre
!
not bet.claimed → ALREADY_CLAIMED
Protection anti double-spend critique
!
bet.choice == winning_option → LOST_BET

Calcul et envoi

reward = split_tokens(total_pool, bet.weight, total_winning_weight) = total_pool × (bet.weight / Σ winning_weights) bet.claimed = True ← mis à jour AVANT sp.send()

Vérifications mathématiques clés

Pool totale : 10+20+30 = 60 XTZ Frais (2%) : 1.2 XTZ → bonus_pool = 1_200_000 mutez Pool nette : 58.8 XTZ → total_pool = 58_800_000 mutez ELO post-résolution (choix 0 gagne) : Alice gagnante 50% → 1000 + 15 = 1015 Bob gagnant 75% → 1000 + 30 = 1030 Charlie perdant 95% → 1000 - 60 = 940

Flux du test complet

1
Vérification de l'état initial (count=0, pool=0, admin correct)
2
Création de 3 prédictions (normale / normale / expirée)
3
Paris d'Alice, Bob, Charlie — vérification comptable de la pool
4
Attaques d'Eve sur place_bet — toutes _valid=False attendues
5
Résolution par admin — vérification des 3 scores ELO
6
Attaques d'Eve sur claim_reward — vérification du claimed=True et double-claim bloqué
